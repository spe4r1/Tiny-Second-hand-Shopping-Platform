| 내 상점 | 본인 판매 상품과 찜한 상품 조회 | 통과 |

## 보안 점검

| 취약점 | 테스트 방법 | 방어 결과 |
| --- | --- | --- |
| SQL Injection | 검색어와 로그인 값에 `' OR 1=1 --` 입력 | 파라미터 바인딩으로 쿼리 구조 변경 안 됨 |
| Stored XSS | 상품명에 `<script>alert(1)</script>` 등록 | HTML 이스케이프되어 텍스트로 표시 |
| Reflected XSS | 검색어에 HTML 태그 입력 | 검색 입력값이 이스케이프되어 출력 |
| CSRF | CSRF 토큰 없이 상품 등록 POST 요청 | 403 권한 오류 반환 |
| IDOR | 다른 사용자의 상품 수정 URL 직접 호출 | 403 권한 오류 반환 |
| Broken Access Control | 본인 상품 장바구니 담기 시도 | 요청 거부 |
| Brute Force | 15분 내 로그인 8회 초과 실패 | 로그인 잠금 메시지 반환 |
| Weak Password Storage | DB의 비밀번호 컬럼 확인 | 평문 없음, PBKDF2 해시만 저장 |
| Session Hijacking 완화 | 쿠키 속성 확인 | HttpOnly, SameSite, Max-Age 적용 |
| Unsafe URL Scheme | 이미지 URL에 `javascript:` 입력 | 입력 검증 실패로 저장 거부 |
| Clickjacking | 응답 헤더 확인 | X-Frame-Options: DENY, CSP frame-ancestors 적용 |
| MIME Sniffing | 응답 헤더 확인 | X-Content-Type-Options: nosniff 적용 |

## 자동 스모크 테스트

```powershell
python -m py_compile .\app.py
python .\tests\smoke_test.py
```

확인 항목:

- 메인 페이지 200 응답
- CSP 보안 헤더 존재
- 회원가입 성공
- 로그인 성공 및 세션 쿠키 발급
- CSRF 토큰 없는 상품 등록 요청 차단
- CSRF 토큰 있는 상품 등록 성공
- XSS 페이로드가 실행 가능한 태그가 아닌 이스케이프 문자열로 출력
- 다른 사용자의 상품 수정 차단
