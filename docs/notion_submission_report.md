# Tiny Second-hand Shopping Platform 개발 보고서

## 0. 프로젝트 개요

본 프로젝트는 시큐어코딩 과제 요구사항을 만족하기 위해 개발한 작은 중고거래 상점 플랫폼이다. 사용자는 회원가입 후 상품을 등록하고, 다른 사용자의 상품을 검색, 찜하기, 장바구니 담기, 구매 확정, 상품 문의까지 수행할 수 있다.

개발 목표는 단순히 기능을 구현하는 것이 아니라, 실제 웹 서비스에서 자주 발생하는 보안 약점인 인증/인가 오류, SQL Injection, XSS, CSRF, 세션 탈취, 무차별 로그인 시도, 위험한 입력값 저장 문제를 고려하여 최대한 안전하게 구현하는 것이다.

- 프로젝트명: Tiny Second-hand Shopping Platform
- 개발 언어: Python 3
- 웹 서버: Python 표준 라이브러리 `http.server`
- 데이터베이스: SQLite
- 프론트엔드: HTML, CSS, JavaScript
- 외부 패키지: 없음
- GitHub Repository: https://github.com/spe4r1/Tiny-Second-hand-Shopping-Platform

### 1.3 보안 요구사항

보안 관점에서 다음 요구사항을 추가로 정의하였다.

- 비밀번호는 평문으로 저장하지 않는다.
- 로그인 세션은 예측 불가능한 난수 토큰으로 생성한다.
- DB에는 세션 토큰 원문을 저장하지 않는다.
- 인증된 사용자만 상품 등록, 찜, 장바구니, 주문, 문의 기능을 사용할 수 있다.
- 판매자 본인만 상품을 수정하거나 판매 상태를 변경할 수 있다.
- 타인의 상품 수정 URL을 직접 호출해도 차단되어야 한다.
- 모든 상태 변경 요청은 CSRF 토큰으로 검증한다.
- SQL 쿼리는 문자열 결합이 아니라 파라미터 바인딩을 사용한다.
- 사용자 입력은 저장 전 길이, 형식, 범위를 검증한다.
- 화면에 출력되는 사용자 입력값은 HTML 이스케이프한다.
- 이미지 URL은 `http`와 `https`만 허용한다.
- 보안 응답 헤더를 적용한다.
- 로그인 실패 횟수를 제한해 무차별 대입 공격을 완화한다.

## 2. 시스템 설계

### 2.1 전체 구조

프로젝트는 외부 의존성을 최소화하기 위해 Python 표준 라이브러리만 사용하였다. 실행 시 SQLite 데이터베이스가 자동 생성되며, 데모 계정과 예시 상품도 함께 생성된다.

```text
tiny-secondhand-secure/
  app.py                         # 웹 서버, 라우팅, 보안 로직, DB 처리
  static/
    style.css                    # 화면 스타일
    app.js                       # 버튼 처리 등 간단한 UI 스크립트
  tests/
    smoke_test.py                # 자동 스모크 테스트
  docs/
    test_checklist.md            # 기능/보안 체크리스트
    secure_coding_report.md      # PDF 생성용 보고서 원본
    notion_submission_report.md  # 노션 제출용 보고서
  scripts/
    build_report_pdf.py          # 보고서 PDF 생성 스크립트
  output/pdf/
    [WHS][secure-coding][XX반]이름(0000).pdf
  README.md
  requirements.txt
```

## 3. 구현

### 3.1 회원가입

회원가입 시 아이디, 비밀번호, 표시 이름, 연락처를 입력한다. 아이디는 영문, 숫자, 밑줄만 허용하며 3-24자로 제한하였다. 비밀번호는 10자 이상이며 영문과 숫자를 포함해야 한다.

비밀번호는 평문으로 저장하지 않고 다음 방식으로 저장하였다.

- 사용자별 16바이트 salt 생성
- PBKDF2-SHA256 사용
- 210,000회 반복
- 저장 형식: 알고리즘, 반복 횟수, salt, hash

### 3.2 로그인과 세션

로그인 성공 시 예측 불가능한 난수 세션 토큰을 생성한다. 이때 DB에는 세션 토큰 원문이 아니라 SHA-256 해시만 저장한다. 쿠키에는 다음 속성을 적용하였다.

- `HttpOnly`: JavaScript에서 세션 쿠키 접근 방지
- `SameSite=Lax`: 기본적인 CSRF 위험 완화
- `Path=/`: 전체 사이트에서 사용
- `Max-Age`: 세션 만료 시간 지정

또한 로그인 실패 기록을 `login_attempts` 테이블에 저장하고, 15분 내 8회 이상 실패하면 추가 로그인을 차단하도록 구현하였다.

### 3.3 상품 등록 및 수정

상품 등록 시 다음 값을 입력한다.

- 상품명
- 가격
- 카테고리
- 상품 상태
- 거래 지역
- 이미지 URL
- 상품 설명

입력값은 저장 전에 길이와 형식을 검증한다.

- 상품명: 2-80자
- 설명: 10-1500자
- 가격: 100원 이상 1억원 이하
- 거래 지역: 2-80자
- 이미지 URL: `http` 또는 `https`만 허용
- 카테고리와 상품 상태: 서버에 정의된 허용 목록만 가능

상품 수정과 상태 변경은 `seller_id == current_user.id` 조건을 검사하여 판매자 본인만 가능하도록 하였다.

### 3.4 상품 검색 및 필터

메인 페이지에서 상품명, 설명, 지역 기준으로 키워드 검색을 할 수 있다. 카테고리와 판매 상태 필터도 제공한다. 검색 쿼리는 SQL 문자열 결합이 아니라 파라미터 바인딩을 사용해 SQL Injection을 방어하였다.

### 3.5 찜하기

로그인 사용자는 타인의 상품을 찜할 수 있다. 본인 상품은 찜할 수 없도록 차단하였다. 같은 상품을 다시 요청하면 찜이 취소된다.

### 3.6 장바구니와 주문

구매자는 타인의 판매중 상품만 장바구니에 담을 수 있다. 구매 확정 시 주문이 생성되고, 주문에 포함된 상품은 판매완료 상태로 변경된다.

적용한 검증은 다음과 같다.

- 로그인 사용자만 장바구니 사용 가능
- 본인 상품 장바구니 담기 차단
- 판매중 상태가 아닌 상품 장바구니 담기 차단
- 구매 확정 시 구매 가능한 상품만 주문 처리

### 3.7 상품 문의

구매자는 판매자에게 상품 문의를 보낼 수 있다. 문의 내용은 2-500자로 제한하였다. 문의 내역은 관련 사용자만 볼 수 있도록 제한하였다.

- 비로그인 사용자는 문의 작성 불가
- 판매자는 본인 상품에 달린 문의 조회 가능
- 문의 작성자는 자신이 보낸 문의 조회 가능
- 관련 없는 사용자는 타인의 문의 조회 불가

### 3.8 보안 헤더

모든 HTML 응답에 다음 보안 헤더를 적용하였다.

| 헤더 | 목적 |
| --- | --- |
| `Content-Security-Policy` | 외부 스크립트 실행 제한, XSS 완화 |
| `X-Frame-Options: DENY` | Clickjacking 방어 |
| `X-Content-Type-Options: nosniff` | MIME Sniffing 방어 |
| `Referrer-Policy: same-origin` | Referer 정보 노출 완화 |
| `Permissions-Policy` | 카메라, 마이크, 위치 권한 차단 |

## 4. 개발 과정에서 확인한 보안 약점과 변경 내용

| 보안 약점 | 발생 가능한 문제 | 적용한 변경 |
| --- | --- | --- |
| 평문 비밀번호 저장 | DB 유출 시 모든 계정 탈취 | PBKDF2-SHA256, salt, 반복 해시 적용 |
| 세션 토큰 원문 저장 | DB 유출 시 로그인 세션 탈취 | DB에는 세션 토큰의 SHA-256 해시만 저장 |
| SQL Injection | 인증 우회, 데이터 유출, 데이터 변조 | 모든 SQL에 파라미터 바인딩 적용 |
| Stored XSS | 상품명/설명에 악성 스크립트 저장 | 출력 시 `html.escape` 적용 |
| Reflected XSS | 검색어에 삽입한 스크립트 실행 | 검색어 출력 시 이스케이프 |
| CSRF | 사용자가 모르는 상품 등록, 주문 요청 | 로그인 세션별 CSRF 토큰 검증 |
| IDOR | 타인 상품 수정 URL 직접 호출 | 판매자 본인 여부 검사 |
| Broken Access Control | 본인 상품 구매, 타인 문의 조회 | 기능별 소유자/상태 검사 |
| Brute Force | 비밀번호 무차별 대입 | 로그인 실패 횟수 제한 |
| 위험한 이미지 URL | `javascript:` 스킴 악용 | `http`, `https` URL만 허용 |
| Clickjacking | iframe에 사이트 삽입 후 클릭 유도 | `X-Frame-Options: DENY` 적용 |
| MIME Sniffing | 브라우저가 파일을 잘못 해석 | `X-Content-Type-Options: nosniff` 적용 |
| 과도한 요청 본문 | 서버 메모리 사용량 증가 | 요청 본문 1MB 제한 |

### 5.2 보안 체크리스트

| 취약점 | 테스트 방법 | 결과 |
| --- | --- | --- |
| SQL Injection | 검색어와 로그인 값에 `' OR 1=1 --` 입력 | 쿼리 구조 변경 없음 |
| Stored XSS | 상품명에 `<script>alert(1)</script>` 등록 | HTML 이스케이프되어 텍스트로 표시 |
| Reflected XSS | 검색어에 HTML 태그 입력 | 이스케이프되어 출력 |
| CSRF | CSRF 토큰 없이 상품 등록 POST 요청 | 403 권한 오류 반환 |
| IDOR | 다른 사용자의 상품 수정 URL 직접 호출 | 403 권한 오류 반환 |
| Broken Access Control | 본인 상품 장바구니 담기 시도 | 요청 거부 |
| Brute Force | 15분 내 로그인 8회 초과 실패 | 로그인 차단 |
| Weak Password Storage | DB 비밀번호 컬럼 확인 | 평문 없음, PBKDF2 해시 저장 |
| Session Hijacking 완화 | 쿠키 속성 확인 | HttpOnly, SameSite, Max-Age 적용 |
| Unsafe URL Scheme | 이미지 URL에 `javascript:` 입력 | 저장 거부 |
| Clickjacking | 응답 헤더 확인 | X-Frame-Options와 CSP 적용 |
| MIME Sniffing | 응답 헤더 확인 | nosniff 적용 |

### 5.3 자동 테스트

다음 명령으로 자동 테스트를 실행하였다.

```powershell
python -m py_compile .\app.py
python .\tests\smoke_test.py
```

자동 테스트에서 확인한 항목은 다음과 같다.

- 메인 페이지 200 응답
- CSP 보안 헤더 존재
- 회원가입 성공
- 로그인 성공 및 세션 쿠키 발급
- CSRF 토큰 없는 상품 등록 요청 차단
- CSRF 토큰 있는 상품 등록 성공
- XSS 페이로드가 실행 가능한 태그가 아니라 이스케이프된 문자열로 출력
- 다른 사용자의 상품 수정 차단

테스트 결과:

```text
smoke tests passed
```

## 6. 유지보수 계획

### 6.1 운영 환경 적용 시 개선할 점

현재 과제용 구현은 로컬 실행과 보안 개념 검증에 초점을 두었다. 실제 운영 환경으로 확장할 경우 다음 보완이 필요하다.

- HTTPS 적용 후 쿠키에 `Secure` 속성 추가
- 서비스 전용 `SECRET_KEY` 기반 세션 서명 추가
- 이미지 업로드 기능을 추가할 경우 확장자, MIME, 용량, 악성 파일 검사 추가
- 관리자 페이지 추가 시 역할 기반 접근 제어 적용
- 주문/결제 기능은 실제 결제 모듈과 서버 검증 콜백으로 분리
- 로그 모니터링과 이상 행위 탐지 추가
- 정기적인 보안 테스트와 코드 리뷰 수행

### 6.2 유지보수 체크포인트

- 신규 기능 추가 시 인증이 필요한 기능인지 먼저 확인한다.
- 사용자 ID, 상품 ID, 주문 ID를 직접 받는 기능은 항상 소유자 검사를 추가한다.
- 새로운 입력값은 길이, 형식, 허용 목록을 검증한다.
- 화면에 출력되는 모든 사용자 입력값은 이스케이프한다.
- SQL 작성 시 문자열 결합을 사용하지 않는다.
- POST, 상태 변경, 삭제, 주문 요청에는 CSRF 검증을 유지한다.
- 보안 헤더가 모든 응답에 적용되는지 확인한다.

Tiny Second-hand Shopping Platform은 과제 요구사항에 맞춰 회원가입, 로그인, 상품 등록, 검색, 찜하기, 장바구니, 주문, 문의, 내 상점 기능을 포함하도록 구현하였다. 또한 비밀번호 해시, 서버 세션, CSRF 토큰, SQL 파라미터 바인딩, HTML 이스케이프, 권한 검사, 로그인 시도 제한, 보안 헤더 등 웹 서비스에서 필요한 기본 보안 대책을 적용하였다.
